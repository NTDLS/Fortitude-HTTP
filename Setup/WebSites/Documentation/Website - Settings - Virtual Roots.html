Website - Settings - Virtual Roots

Virtual roots allow a server administrator to build virtual directory structures under a website. In this way a website can have its root located in "C:\Websites\My Website" but then have requests for "/images" locally and silently redirected to "C:\CommonImages\".

Virtual Roots

Allows the server administrator to add, edit, delete and reorder website virtual roots.


Virtual Roots - Item




Setting Description
Web path The relative path from the website root which is to be interpreted as an alternate physical path.
File system path The local or UNC path which the request should be locally redirected to.
Authentication required on resource Defines whether the website virtual root path requires credentials other than that of the service account under which the server is running.
Username The username of the user to impersonate when connecting to the website virtual root path.
Domain The domain of the user to impersonate when connecting to the website virtual root path.
Password The password of the user to impersonate when connecting to the website virtual root path.
